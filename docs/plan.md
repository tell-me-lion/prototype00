1. 정규식(Regex) 기반 1차 클렌징
   가장 저렴하고 확실한 방법, 파이썬의 re 모듈을 사용해 기계적인 노이즈를 제거

타임스탬프 및 화자 ID 제거: r"<\d{2}:\d{2}:\d{2}>\s[a-f0-9]+:\s" 패턴으로 앞부분을 통째로 날리기

단순 불용어(Stopwords) 제거: "자", "여러분", "오늘", "일단은" 같은 상투적인 강의 시작어들을 사전에 정의해 필터링

2. 형태소 분석기(Kiwi) 기반 구어체 문장 분리
   음성 인식(STT) 스크립트의 가장 큰 문제는 마침표가 없거나 엉뚱한 곳에 찍혀 있다는 점

기술: Kiwi (키위) 형태소 분석기. (Mecab보다 구어체와 띄어쓰기 오류, 마침표 복원에 압도적으로 뛰어나다는 평가)

역할: split_into_sents() 함수를 사용해 길게 이어진 텍스트를 논리적인 문장 단위로 쪼갭니다.

품사 기반 필터링: 문장 내에 명사(NNG)나 동사(VV) 없이, 감탄사(IC)나 접속사(MAJ)로만 이루어진 짧은 문장("네 좋습니다.", "됐을까요?")은 파서 단에서 드랍

3. 경량 인코더 모델을 활용한 추출 요약 (Extractive Summarization)
   LLM에게 "문장을 새로 써줘"라고 하는 대신, 있는 문장 중에서 **가치 있는 문장만 분류(Classification)**

기술: KoELECTRA 또는 KR-SBERT 같은 파라미터가 작은 경량 인코더 모델.

역할: \* 지도 학습(SFT) 방식을 빌려, 문장을 입력하면 이 문장이 '핵심 지식(Fact)'인지 '단순 잡담/설명(Noise)'인지 이진 분류(Binary Classification) 하도록 모델을 가볍게 파인튜닝? 또는 모델서치

임베딩 벡터를 뽑아 문장 간의 코사인 유사도(Cosine Similarity)를 계산하여, 주제가 넘어가는 구간을 수학적으로 잘라내는 시맨틱 청킹

4. 로컬 SLM + 구조화 파서 (Fact 구조화)
   전통적 NLP 파서로 정제된 핵심 문장 덩어리들을 최종 JSON 형태로 규격화할 때만 가벼운 생성형 모델

기술: Llama-3-8B-Instruct (한국어 튜닝 버전) 또는 Qwen-7B 등 로컬 GPU 리소스로 구동 가능한 오픈소스 SLM + Pydantic 기반의 구조화 라이브러리(예: Instructor 또는 Outlines).

역할: 프롬프트에 엄격한 JSON 스키마를 걸고, 추출된 텍스트 덩어리를 넘겨 "이 텍스트에서 주어-동사가 명확한 사실(Fact) 명제 3개만 추출해 JSON으로 반환해"라고 지시합니다. SLM은 문맥을 다듬어주는 역할만 수행하므로 환각이 통제됩니다.
