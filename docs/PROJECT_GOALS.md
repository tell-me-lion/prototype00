# 프로젝트 목표 및 평가 기준

## 목표

> **모든 것은 클라이언트 관점에서!**

- 강의 스크립트에서 핵심 개념과 학습 포인트를 자동으로 추출
- 주관식/객관식/빈칸 채우기/코드 실행 등 다양한 형태의 퀴즈와 해설을 생성하는 시스템 구축
  - 퀴즈 생성 전략, 퀴즈 유형, RAG 구조
- 주차별 학습 가이드와 핵심 요약을 자동으로 제작하여 수강생에게 제공

---

## 추가 기능 (보너스)

- 수강생 오답 패턴 분석을 통한 개인 맞춤형 추가 학습 추천
- 퀴즈 난이도 적응형 조절 알고리즘
- LMS 연동을 통한 자동 퀴즈 배포 시스템

---

## 평가 기준

### 핵심 원칙

- 클라이언트 관점에서 서비스의 완성도, 커뮤니케이션 능력, 문제 해결 접근법
- *코드에 대한 상세 리뷰보다는* 서비스가 요구사항에 맞게 잘 만들어지고 있는지, 프로젝트 진행 상황에 대한 커뮤니케이션이 잘 되고 있는지
- 모든 산출물은 클라이언트 보고 형식으로 작성

### 배점

| 평가 항목 | 배점 | 세부 기준 |
|-----------|------|----------|
| 서비스 완성도 | 30% | 퀴즈/학습 가이드 품질, 다양한 퀴즈 유형 지원, UI 완성도 |
| 문제 해결력 | 25% | 퀴즈 품질 보장 방법, 난이도 조절 로직의 합리성 |
| 커뮤니케이션 | 25% | 기획서/보고서 명확성, 진행 상황 공유 적절성 |
| 프레젠테이션 | 20% | 시연 명확성, 실제 교육 현장 적용 가능성 어필 |

## 제출물

### 중간평가 (2주차)

| # | 제출물 | 평가 포인트 (클라이언트 관점) |
|---|--------|------------------------------|
| 1 | 프로젝트 기획서 (퀴즈 유형, 생성 전략, RAG 구조) | 퀴즈가 강의 핵심 내용을 잘 반영하는가? |
| 2 | 핵심 기능 시연 영상 (3분 이내: 1개 강의 기반 퀴즈 생성 시연) | 수강생 관점에서 학습에 도움이 되는가? |
| 3 | 진행 현황 보고서 | 프로젝트 방향성과 진행 상황 소통이 원활한가? |
| 4 | 발표 영상 (5분 이내) — 얼굴 노출 불필요, PT + 음성 설명 녹화로 대체 | — |

### 최종평가 (4주차)

| # | 제출물 | 평가 포인트 (클라이언트 관점) |
|---|--------|------------------------------|
| 1 | 퀴즈 세트 샘플 (3개 강의 이상, 강의당 10문항+) | 퀴즈의 다양성과 품질은 충분한가? |
| 2 | 학습 가이드 샘플 (주차별 요약, 개념 맵) | 해설이 학습에 실질적으로 도움이 되는가? |
| 3 | 서비스 시연 영상 (5분 이내) | 학습 가이드가 수강생 관점에서 유용한가? |
| 4 | 최종 보고서 및 GitHub Repository | 서비스 사용 경험이 직관적인가? |
