문제 정의

수강생의 어려움

  • 강의 녹화본은 쌓이지만 복습이 어렵다
  • 핵심 내용을 직접 정리하는 데 시간이 과다 소요
  • 스스로 학습 수준을 점검할 방법이 부족하다

교육 업체의 한계

  • 강의마다 요약·퀴즈를 수작업으로 만들 리소스 부족
  • 주차별 학습 가이드 제작에 반복적 노동 발생
  • 콘텐츠 품질의 일관성 유지가 어렵다

"강의를 선택하기만 하면, 나머지는 AI가 처리할 수 없을까?"

솔루션 — 알려주사자

두 가지 작동 모드를 통해 단기 복습부터 주간 누적 복습까지 지원합니다.

산출물 주차별 학습 가이드, 핵심 요약

Mode A — 단일 강의 지식 추출

가공된 데이터베이스를 바탕으로 할루시네이션 없는 콘텐츠를 생성합니다.

1. 핵심 개념 (Concept)

개념 정의, 중요도 점수(0~1), 연관 개념 매핑


{
  "concept_id": "concept_추상",
  "concept": "추상",
  "definition": "추상 클래스는 주요 메소드인...",
  "importance": 0.302,
  "related_concepts": ["concept_바이트", "concept_캐릭터"]
}

2. 설계 (Blueprint)

정답 근거 및 오답 재료 사전 분리로 신뢰도 확보


{
  "blueprint_id": "bp_추상",
  "concept": "추상",
  "evidence": {
    "correct_facts": [
      "추상 클래스에는 스트링 단위로 된 애가 없었다."
    ],
    "distractor_facts": [
      "추상 클래스는 character 세트를 가지고 있다.",
      "OutputStream은 추상 클래스이고..."
    ]
  }
}

Mode A — 퀴즈 생성 및 5대 유형

유형 설명
개념 정의형 객관식 핵심 개념의 정의 4지선다
오개념 객관식 흔한 오해를 선지에 포함하여 검증
빈칸 채우기 핵심 용어를 타이핑하는 직접 입력형
OX 퀴즈 참/거짓 판별 방식의 빠른 개념 확인
코드 실행형 SQL/코드의 실행 결과를 예측하는 문제

생성된 퀴즈 예시

난이도: 중 

{
  "question_type": "mcq_definition",
  "question": "추상 클래스에 대한 설명으로 올바른 것은?",
  "choices": [
    { "text": "인스턴스를 직접 생성할 수 있다.", "is_answer": false },
    { "text": "추상 메서드를 포함할 수 있다.", "is_answer": true }
  ],
  "explanation": "하위 클래스가 반드시 구현해야 하는 추상..."
}

Mode B — 주차별 학습 가이드

1주치 강의 전체를 통합 분석하여 학습 로드맵과 핵심 요약을 제공합니다.

통합 산출물 포맷

{
  "week": 5,
  "summary": "이번 주차에서는 Java I/O 스트림의 계층 구조와 직렬화...",
  "key_concepts": [
    "추상 클래스", 
    "바이트 스트림", 
    "직렬화"
  ],
  "meta": { 
    "lecture_count": 4, 
    "total_concepts": 15 
  }
}

통합 가이드의 장점

  • 여러 강의에서 산발적으로 언급된 개념을 주차 단위로 하나로 통합
  • 강의 간의 흐름과 맥락을 학습자에게 로드맵 형태로 제공
  • 한 주 동안 배운 핵심 개념에 대한 효과적인 누적 반복 복습

서비스 흐름

1

대시보드

전체 학습 진행률과 최근 분석 결과를 한눈에 확인

2

Mode 선택

단일 강의(Mode A) 또는 주차별 통합(Mode B) 선택 후 자동 처리

3

단일 분석 확인

추출된 핵심 개념, 학습 포인트, 5개 유형 퀴즈 풀이

4

주차 통합 복습

1주치 요약본과 로드맵으로 체계적인 시험 대비

파일 업로드나 복잡한 설정 없이, 대시보드 클릭만으로 맞춤형 학습 콘텐츠를 획득합니다.

핵심 기술 — 전처리 5단계 데이터 플로우

정제되지 않은 강의 스크립트(raw)를 RAG에 즉시 활용 가능한 구조화된 지식 DB로 변환

Phase 1

세션 분리 및 정제

  • 15초 이내 발화 단락 병합
  • 30분 이상 공백 시 세션 분할
  • Gemini API 기반 문맥 교정
Phase 2

문장 복원 및 필터링

  • 구어체 텍스트 문장 단위 분할
  • 품사(POS) 기반 단순 감탄사 제거
Phase 3

시맨틱 청킹 및 스코어링

  • 문맥 전환 지점 기반 청크 분할
  • TF-IDF로 청크 내 핵심어 추출
Phase 4

지식 명제 추출 및 구조화

  • 패턴 & LLM 기반 '명제(Fact)' 추출
  • 핵심 키워드와 명제의 '개념-설명' 세트화
Phase 5

최종 RAG 팩트 DB 구축

  • 원문/키워드/명제 매핑 (ChunkDoc)
  • 단일 개념으로 정보 병합 (ConceptDoc)

퀴즈 품질 보장 전략

사전 설계(Blueprint)부터 자동 검증까지 이어지는 체계적인 파이프라인

1. Blueprint
개념별 퀴즈 유형·난이도·수량 사전 설계
→
2. Evidence
정답(Correct) 및 오답(Distractor) 근거 확보
→
3. Generation
설계 명세 기반 LLM 문항 자동 생성
→
4. Validator
정확성 및 사실 일치 여부 자동 검증

문항 설계 및 난이도 제어

난이도 질문 길이 및 특징 출제 대상 선지 구성
하 15~24자 (짧은 질문) 기초 문법, 개념어 식별 개념어 위주 (4지선다 허용)
중 21~30자 (단문 서술형) 개념 정의, 특징 판단 명사구, 문장형 (5지선다)
상 30자 이상 (상세 조건형) 복수 조건, 예외 판별 복합 서술형 (5지선다)

출제 Quota

  • O/X 퀴즈: 최대 20%
  • 빈칸 채우기: 최소 20% 이상
  • 객관식: 오답근거 2개 이상

오답 선지 설계 4대 패턴

1. 인접 개념 함정 유사 개념의 특징 차용
2. 속성 반전 함정 핵심 속성을 반대 특성으로 교체
3. 키워드 중첩 함정 정답 키워드 + 성질 조작
4. 과장형 함정 "반드시", "~에서만" 절대 표현

자동 검증 (QA Validator)

LLM 생성 퀴즈를 4가지 기준으로 검증하여 통과한 문항만 최종 적재

① 근거성 (Evidence)

≥ 0.30

문항이 강의 원문에 기반하는가?

출처 텍스트 ↔ 검증 청크 TF-IDF 점수

② 중복성 (Uniqueness)

< 0.85

유사한 내용을 묻는 문항이 없는가?

문항 간 내용 유사도 스코어

③ 난이도 (Difficulty)

0.2 ~ 0.95

너무 쉽거나 무의미하게 어렵지 않은가?

LLM 예측 기반 오답률 허용 범위

④ 변별력 (Discrim.)

설득력≥0.4

오답이 소거법으로 쉽게 풀리지 않는가?

오답 설득력 스코어 및 소거 가능 비율 판단

※ 검증 실패 문항은 제외되며, 전체 통과율과 실패 사유가 자동 로깅됩니다.
